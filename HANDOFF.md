# BusTrack — Session Handoff

> 最後更新：2026-07-13（Claude session）

## 專案是什麼

多路線、多目的地的「公車通勤不遲到」PWA。手動輸入「幾分鐘到站＋車程＋步行」，
算出實際抵達時刻，跟該目的地的最晚抵達時間比對（準時／剛好趕上／會遲到）。
推薦班次＝還沒進站、走得到、趕得上者中「抵達時間最貼近截止時間」的那班。

純靜態、無 build step：`index.html`（全部邏輯內嵌）＋ `manifest.webmanifest` ＋ `sw.js`。

  點 active chip＝單一編輯、＋chip 新增、v1 資料自動遷移
- 齒輪＝「所有目的地」管理面板：全部名稱/時間就地編輯、列刪除（至少留一）、

- 「走到站牌（分鐘）」欄位：出門提示用 leaveIn＝進站倒數−走到站牌
  （≥10 還不用出門／4–9 準備出門／≤3 現在出門！）；推薦排除走路來不及的班次，
  卡片標紅字「走過去來不及」
- 「更新」modal 精簡：只顯示到站分鐘欄位（其他欄位隱藏保值）

- **藍色 Hero 卡**（Transit App 風，2026-07-13 完成）：
  頂列（目的地・期限・現在時間＋白底出門提示 pill）→ 大字倒數（數字 2.6rem）
  ＋右欄抵達時刻/淡綠提早文字 → 路線牌＋上車時間 → 站牌列（上車→下車）→
  時間軸（🚶出門→🚌上車→●抵達，上車節點固定 35%，出門時刻＝上車−走到站牌，
  未填走到站牌退回顯示「現在」）→ glass footer（車程・步行・↻更新時刻）
- **清單卡片改版**（2026-07-13 完成）：移除底部「更新／編輯」按鈕列——
  更新改為倒數文字旁的 28px 圓形 ↻ 鈕（事件委派優先於卡片點擊），
  點整張卡片任意處＝開編輯 modal（.card 有 data-id、:active 縮小回饋）；
  副標精簡為「預計 HH:MM 進站」一句；pill 固定帶「站牌走 N 分・」前綴
  （未填走到站牌則省略），文案：綠「早到 N 分到目的地」（≥8 分）、
  黃「只比預計時間早 N 分到目的地」（0–7 分）、紅「會晚 N 分到目的地」；
  走不到的班次（未進站且 leaveIn<0）紅 pill「走過去來不及」取代原本
  矛盾的綠色準時 pill

所有功能都在瀏覽器面板用模擬操作驗證過（含 touch 手勢、遷移、刪除流程）。
